Java NIO

Java NIO Buffer

Buffer基础用法

使用 Buffer 读写数据通常需要以下四步：

  1. 将数据写入Buffer
  2. 调用buffer.flip()
  3. 从Buffer中读取数据
  4. 调用clear()或者compact()
    当将数据写入一个buffer时，这个buffer会监测写入了多少数据。一旦需要读取写入的数据时，需要调用flip()将buffer从写模式转换到读模式。
    当读取了所有buffer中的数据后，为了能使buffer可以再次被写入，需要清空buffer。有两种方式可以实现：调用clear()或者compact()。clear()方法会清除整个buffer。compact()方法仅仅清除已经被读取的数据，而未被读取的数据会被移动到buffer头部，数据将会被写入未被读取数据的后面。

Buffer Capacity, Position and Limit

buffer在本质上是一个可以写入数据并且再读出的内存块。这块内存被包装成一个NIO Buffer对象，提供了一系列的方法使得我们可以很容易的操作这块内存。
每个Buffer拥有三个重要的属性：

  1. capacity
  2. position
  3. limit
    position以及limit的意义在于Buffer处于写模式或者读模式，Capacity则不区分读或写模式。

Capacity

作为一块内存，Buffer有确定的大小，也叫作容量。可以写入容量范围内的bytes，longs，chars等类型数据到buffer。一旦Buffer填满，如果需要继续写入，则必须将buffer置空（读取数据或者清除数据）。

Position

在写入数据到Buffer时，要明确写入位置。初始时position是0，当写入数据时position回进而指向下一个位置使得下一组数据插入。Position最大值为capacity-1。
当从buffer中读数据时，同样需要一个给定的position，并且在读取一次后position指向下一个位置使得再次读取。当翻转buffer从写模式转为读模式时，position将被重置为0。

Limit

当处于写模式是，buffer的limit表示有多少数据可以被写入缓冲区，写模式下limit的值等于buffer的capacity。
当翻转为读模式时，limit表示有多少数据可以读取。因此，当翻转buffer到读模式后，limit被设置为写模式下的position。换而言之，写入多少字节便可以读取多少(limit被设置为写入的数量，写入数量则有position标记)。

Buffer Types

Java NIO有一下集中缓冲区类型：

  • ByteBuffer
  • MappedByteBuffer
  • CharBuffer
  • DoubleBuffer
  • FloatBuffer
  • IntBuffer
  • LongBuffer
  • ShortBuffer
